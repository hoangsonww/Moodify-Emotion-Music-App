About the Project

What is Moodify?

An intelligent music recommendation system powered by cutting-edge AI/ML technology

🎯

Mission

To revolutionize music discovery by creating an intelligent system that understands and responds to human emotions through advanced AI technology.

🚀

Vision

Providing personalized music recommendations that perfectly match users' emotional states, detected through text, speech, and facial expressions.

💡

Innovation

Combines three self-trained AI/ML models (BERT, SVC+MFCC, FER) with Deezer's free Search API, served from a memory-snapshot Modal container that scales to zero when idle.

🌐

Accessibility

Available on web and mobile platforms with progressive web app features, ensuring a seamless experience across all devices.

Layered Architecture

  • Client Layer: React web app and React Native mobile app for user interaction
  • Load Balancer: NGINX for distributing traffic and SSL termination
  • Application Layer: Django REST Framework backend and Flask ML services
  • AI/ML Layer: Three specialized models for emotion detection
  • Data Layer: MongoDB for persistence, Redis for caching, SQLite for development
  • Analytics Layer: Apache Spark and Hadoop for big data processing

Request Flow

  1. User submits emotion input through web or mobile interface
  2. Request is load-balanced by NGINX to backend instances
  3. Backend checks Redis cache for existing results
  4. If cache miss, forwards to AI/ML service for emotion detection
  5. ML model processes input and returns emotion with confidence score
  6. Inference service queries Deezer Search API for mood-matched tracks
  7. Results are cached in Redis and stored in MongoDB
  8. Response is returned to user with recommendations

⚠️ Important Notes

  • The Django API is deployed to Vercel and the ML inference service runs on Modal with memory snapshots + scale-to-zero, so the first request after idle may take ~1-2 s while a container restores. Subsequent calls hit a warm container.
  • Facial and speech emotion detection models load on the first cold start; once snapshotted they restore in seconds rather than re-loading from disk.
  • For optimal performance, consider cloning the repository and running locally.
  • Modal scales to zero when idle, so usage cost is essentially free for a personal-traffic app.

🛠 How It Works

From Mood to Music in Five Steps

A single API hop hides three models, a Deezer search, and personalised re-ranking.

01
✍️

Share a moment

Type a few words, record a clip, or snap a selfie. Your input never leaves the device except for the single inference call.

02
🧠

Detect the emotion

One of three models classifies your input. Text → BERT. Voice → wav2vec-style. Face → FER + facenet-pytorch.

03
🎯

Personalise

Your recent mood history blends in at an affinity-driven rate; an optional user-picked genre biases the search phrase.

04
🎶

Tune the music

The recommender queries Deezer's keyless search endpoint and returns up to 60 de-duplicated, quality-ranked tracks.

05
📜

Track + learn

Every preview play + Deezer open feeds back into history so the next pass gets sharper.

❓ FAQ

Frequently Asked Questions

Quick answers to what people ask most.

How does Moodify detect my mood?

Three self-trained models: a BERT classifier reads tone + sentiment from typed text, a wav2vec-style network reads pace + energy from a short voice recording, and an FER + facenet-pytorch pipeline reads your expression from one front-camera photo.

Where do the recommendations come from?

Deezer's free, keyless search endpoint. The recommender prepends a mood-tuned keyword phrase (optionally biased by genre + your recent mood history), returns up to 60 de-duplicated tracks per analysis, and pages the result client-side.

Does Moodify store my photos or voice clips?

No. Photos and voice recordings are sent to the inference service for a single classification call and are never persisted server-side. Only the resulting mood label + saved track metadata are stored in your profile.

Is Moodify free?

Yes. The web client runs on Vercel (free tier), the ML inference runs on Modal with scale-to-zero billing, and Deezer search is keyless. End-user cost is $0.

What stack does the app use?

React 18 + MUI v5 for web, Expo SDK 51 + React Native 0.74 for mobile, Django 5.1 + DRF on Vercel for the API, MongoDB Atlas for storage, and FastAPI on Modal for the three-model inference service.

Why Deezer instead of Spotify?

Spotify locked down `/v1/search` for client-credentials apps; every request 403s. Deezer's keyless search is reliable, includes 30-second previews, and deep-links to a streamable result page without OAuth.

Can I self-host?

Yes. The repository ships Docker Compose, Helm charts, Terraform modules, and Modal app definitions. See `DEPLOYMENT.md` and `INFRASTRUCTURE_SETUP.md`.

What's on the roadmap?

Spotify playback link, in-app preview player, mood streaks, social mood-sharing, Apple/Google Sign-in, push notifications for weekly mood wraps, and an Apple Watch companion.
